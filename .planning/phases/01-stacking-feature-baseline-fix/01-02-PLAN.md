  <action>
Create `scripts/backward_elimination.py` that:

1. Loads data with `load_data(feature_level="v96624_plus")`
2. Starts with all 21 features from `FEATURES_V96624_PLUS`
3. For each iteration:
   a. Run full pipeline CV with current feature subset + LR head config
      (head_model="logit", head_base_only=True, calibration_mode="none")
   b. Compute permutation importance: for each feature, shuffle it in training data,
      re-run CV, measure Hybrid Score drop via `hybrid_score()`
   c. Identify feature with smallest importance (least score drop when shuffled)
   d. If removing it does not hurt Hybrid Score by more than 0.0005, drop it
   e. Print: iteration number, feature dropped, score before/after, remaining count
4. Stop when any removal hurts score by > 0.0005 or only 10 features remain
5. Print final ranked list and score curve

Key details:
- Use existing CV loop pattern (5x3 RepeatedStratifiedKFold matching train.py)
- Permutation importance operates at BASE MODEL level (shuffle input features
  before RSF/EST/XGBCox training), not at head level
- Use n_repeats=3 for importance ranking (speed), full repeats for final confirm
- Print progress so user can monitor
