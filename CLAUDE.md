# WiDS 2026 Kaggle Competition

## 实验管理规则

1. 每次代码改动后运行 `python -m src.train` 验证 CV 指标
2. 每次 Kaggle 提交后，用户提供 LB 分数时，必须在 experiments.md 详细记录
3. 每次重大改动创建 git commit
4. 保持工作目录整洁，不创建临时文件
5. 每次代码改动如果有 CV 提升，立即提交 git commit
6. 每次用户提供 Kaggle LB 分数时，立即更新 experiments.md 记录分数
7. 详细状态、发现、下一步计划见 `experiments.md` 末尾的"当前状态"章节
