1. 项目总览

这是什么?

这是一个参加 WiDS Global Datathon 2026 Kaggle 竞赛的项目。任务：根据山火发生后前 5 小时的观测数据，预测火灾在 12h / 24h / 48h / 72h 内烧到 5 公里疏散区的概率。

数据规模

极小数据集：训练集 221 个样本，测试集 95 个样本，34 个原始特征。

这意味着传统树模型比深度学习更合适，降低方差（多种子平均）比调参更重要。

2. features.py -- 特征工程

这个模块是整个管道的"数据厨房" -- 把 34 个原始火灾特征加工成模型能用的输入。分三个阶段：

阶段一：去冗余

删除 10 个相关系数 |r| > 0.95 的冗余特征。示例：

删除保留相关系数
relative_growth_0_5harea_growth_rel_0_5hr = 1.0
projected_advance_m-dist_change_ci_0_5hr = -1.0
centroid_displacement_mradial_growth_mr ~ 0.99
log1p_growtharea_growth_abs_0_5hr ~ 0.97

结果：34 → 24 个特征。对于使用 max_features="sqrt" 的树模型，去冗余能提高每次分裂时选中真正有用特征的概率。

阶段二：特征工程 (add_engineered)

第一轮：基础物理量

特征公式物理含义
log_distlog1p(距离)压缩距离的长尾分布
dist_area_ratio距离 / sqrt(面积)距离近 + 火大 = 威胁高

第二轮：重叠区特征 (657m ~ 4545m)

这个距离范围内的样本最难区分（有的烧到了，有的没有）。这些特征专门针对这个"模糊地带"：

hour_sin/cossin/cos(小时 / 24)圆形时间编码（23点和0点是相邻的）

第三轮：交互特征

特征公式物理含义

directional_threat面积 * 对齐度大火 + 对准社区 = 方向性威胁
expansion_threat径向增长 * (1/距离)快速扩张 + 近距离 = 扩张威胁

阶段三：特征集选择

Insight

eta_hours（预估到达时间）是排列重要性最高的工程特征（CI_drop=0.00783），因为它直接编码了"火什么时候到"这个核心问题，比单独看距离或速度更有信息量。

medium 胜出的原因：在 221 个样本上，过多特征会引入噪声。36 个特征是信噪比的最佳平衡点 -- 足够多的交互/重叠区特征来帮助模型，又不至于过拟合。

3. models.py -- 模型架构

7 种生存分析模型，共享统一接口：fit(X, y_time, y_event) 和 predict_proba(X) 返回 {12: [...], 24: [...], 48: [...], 72: [...]}。

阵营 A：生存函数模型（连续时间）

学习一条完整的生存曲线 S(t)，然后在 12/24/48/72h 四个时间点上取值。

模型库核心假设状态
CoxPHlifelines比例风险（形状相同，倍数不同）可用
WeibullAFTlifelines风险单调递增可用
LogNormalAFTlifelines风险先升后降可用
RSFsksurv无参数假设，让数据说话主力
ESTsksurv同 RSF，分裂点随机选取主力
GBSAsksurv梯度提升生存分析失败 (#10)

阵营 B：逐时间窗口分类模型（离散时间）

每个时间窗口当成独立的二分类问题："H 小时内烧到了吗？是/否"

模型库状态
MultiHorizonXGBxgboost可用
RankXGBxgboost可用
MultiHorizonLGBMlightgbm可用
MultiHorizonCatBoostcatboost可用

RSF vs EST：当前最佳组合

RSF：每个节点从 sqrt(n) 个候选特征中选最优分裂点（贪心搜索）。

EST：每个节点从 sqrt(n) 个候选特征中随机选分裂点（不搜索最优）。

EST 单棵树更弱，但集成后方差更低。两者组合提供真正的模型多样性 -- 实验 #9 证明了这一点（LB +0.00122）。

关键实现：_sf_to_probs

def _sf_to_probs(surv_fns, horizons):
    for h in horizons:
        for i, fn in enumerate(surv_fns):
            # 边界处理，与原始 0.96624 代码对齐
            s_val = fn(h) if h <= fn.x[-1] else fn(fn.x[-1])
            p_arr[i] = 1.0 - s_val
    return result

将阶梯形生存函数转换为时间窗口概率。边界处理 fn(fn.x[-1]) 是与参考 0.96624 代码对齐的关键。

Insight

阵营 A（生存函数）完胜阵营 B（逐窗口分类），因为生存曲线天然保证时间单调性（S(t) 递减 → P(t) 递增），而独立分类器需要额外后处理来强制单调。

GBSA 在实验 #10 中失败，因为其 12h 概率被压缩到极窄范围（0.212-0.268）。C-Index 对"近似平局"极其敏感 -- 微小的排序错误就会导致分数崩溃。

Insight

5. evaluation.py -- 评分体系

混合分数公式

Hybrid = 0.3 * C-Index + 0.7 * (1 - WeightedBrier)

C-Index（权重 30%）-- 排序能力

输入：仅 prob_12h

问题：对于任意一对样本 (A, B)，如果 A 比 B 先烧到社区，模型是否给 A 分配了更高的风险分数？

关键特性：只关心相对排序。所有概率乘以 2，C-Index 不变。

范围：0.5 = 随机猜，1.0 = 完美排序

ci = lifelines_cindex(y_time, -risk_scores, y_event)
# 取负号是因为 lifelines 约定"越大 = 生存越久"
# 但 prob_12h 的含义是"越大 = 越危险 = 生存越短"

加权 Brier Score（权重 70%）-- 校准精度

输入：prob_24h、prob_48h、prob_72h（不含 12h！）

公式：WeightedBrier = 0.3 * Brier@24h + 0.4 * Brier@48h + 0.3 * Brier@72h

Brier Score：mean((预测概率 - 真实标签)^2) -- 越低越好

标签构建规则（每个时间窗口 H）

条件标签解释
event=1, time ≤ H1火在 H 小时内烧到了
event=1, time > H0烧到了但在 H 之后
event=0, time ≥ H0观测到 H 都没烧到
event=0, time < H排除审查：在 H 之前失去追踪，结果未知

评分流程图

prob_12h ─────────────────────> C-Index (排序) ──> x0.3 ─┐
                                                            ├──> Hybrid
prob_24h ──> Brier@24h ──> x0.3 ─┐                        │
prob_48h ──> Brier@48h ──> x0.4 ─┼─> WeightedBrier        │
prob_72h ──> Brier@72h ──> x0.3 ─┘   → (1-WB) → x0.7 ─┘
Insight

C-Index 和 Brier 可能冲突：C-Index 只要排序对（你可以把所有概率压缩到 [0.01, 0.02] 范围内，C-Index 照样满分），但 Brier 要求绝对值准确。GBSA 在实验 #10 中的失败就是这个矛盾的极端体现。

72h Brier 权重 0.3 但贡献几乎为零，因为 prob_72h=1.0 且几乎所有标签也是 1。真正的战场在 24h 和 48h。

审查样本的排除机制意味着每个时间窗口的 Brier 是在不同子集上计算的。48h 的有效样本最多，所以权重最高（0.4）。

6. monotonic.py -- 后处理

最后一道关卡：确保输出的概率"符合物理常识"。

submission_postprocess（提交专用）

# 第一步：12h 独立处理（仅用于 C-Index 排序）
result[12] = clip(prob_dict[12], 0.01, 0.99)

# 第二步：24h/48h/72h 单独的单调链
prev = prob_dict[24]
result[48] = max(prob_dict[48], prev)    # 48h ≥ 24h
result[72] = max(prob_dict[72], result[48])  # 72h ≥ 48h

# 第三步：72h 硬编码为 1.0
result[72] = 1.0

# 第四步：裁剪 24h/48h
result[24] = clip(result[24], 0.01, 0.99)
result[48] = clip(result[48], 0.01, 0.99)

关键设计决策

决策原因
12h 与 24h+ 解耦12h 只影响 C-Index（排序），24h+ 只影响 Brier（校准）。强制 24h ≥ 12h 可能扭曲 24h 的校准精度。
裁剪到 [0.01, 0.99]Brier 惩罚是二次的。裁剪把最坏情况从 (0-1)^2=1.0 限制到 (0.01-1)^2=0.98。C-Index 不受影响（排序不变）。
72h = 1.0 硬编码几乎所有符合条件的火灾在 72h 内都会到达疏散区。预测 1.0 能最小化 Brier。72h 不影响 C-Index。

enforce_monotonicity vs submission_postprocess

enforce_monotonicitysubmission_postprocess
12h 处理参与单调链独立裁剪
单调链12h → 24h → 48h → 72h仅 24h → 48h → 72h
72h保持模型输出硬编码 1.0
裁剪范围[0, 1][0.01, 0.99]
用途通用工具竞赛提交专用
Insight

12h 与 24h+ 解耦看似微妙但很重要。实验 #4 尝试了 logit 空间校准，把 12h 中位数从 0.015 提升到 0.067，但 LB 反而下降了 0.012，因为校准扭曲了 24h/48h 的绝对值。

这个模块是整个管道中唯一"注入领域知识"的地方（72h=1.0 不是模型学出来的，是人类根据数据分布做的判断）。这种"模型 + 规则"的混合策略在小数据竞赛中很常见。

核心结论

1. 特征集 > 超参数：切换到 MEDIUM 特征集带来 +0.00222，而调 max_features 反而 -0.00038。

2. 模型多样性 > 更多树：加入 EST（不同分裂策略）带来 +0.00122，而 n_estimators 从 200 增到 1000 收益甚微。

3. 别动有效的东西：Logit 校准（实验 #4）和逐窗口拆分（实验 #10）都导致了严重回退。

Insight

最引人注目的规律：CV 分数几乎不动（0.9690-0.9724），但 LB 从 0.94688 到 0.96331 波动巨大。这意味着微小的泛化差异在排行榜上被放大了。在 95 个测试样本上，哪怕 1-2 个样本的排序变化就能让分数波动 0.001 以上。

8. 全部参数详解

整个项目涉及 60+ 个可调参数，分布在配置、模型、特征工程、后处理和评分五个层面。以下逐一说明。

8.1 交叉验证参数 (config.py)

参数值含义
N_SPLITS5K-Fold 的 K，每次重复分 5 折
N_REPEATS10重复次数，总共 5x10=50 轮训练
RANDOM_STATE42随机种子，保证 CV 折划分可复现
HORIZONS[12, 24, 48, 72]预测的 4 个时间窗口（小时）

为什么 5 折 x 10 次？221 个样本太少，单次 5 折的方差很大。重复 10 次用不同的随机划分，每个样本被预测 10 次取平均，OOF 估计更稳定。

8.2 RSF 参数 (models.py)

参数类默认值提交实际用值含义
n_estimators1000200森林里树的数量。CV 用 1000（单模型需要足够多树），提交用 200（10 个模型平均，总树数=2000）
max_depth55每棵树最大深度。221 样本上深度 5 已足够，再深就过拟合
min_samples_leaf55叶节点最少样本数，约占训练集的 2.3%，防止叶子太细碎
min_samples_split1010节点分裂最少样本数，低于 10 个样本就不再分了
max_features"sqrt""sqrt"每次分裂随机选 sqrt(36)≈6 个候选特征。实验 #7 试过 12，LB 反而降了
random_state425 个种子轮换[42, 123, 456, 789, 2026]，每个种子产生不同的随机森林
n_jobs-1-1用所有 CPU 核心并行训练
Insight

max_features="sqrt" 是 RSF 的核心正则化手段。36 个特征里每次只看 6 个，迫使不同树关注不同特征子集，降低树之间的相关性。实验 #7 把它改成 12，CV 微涨 +0.0003 但 LB 跌了 -0.00038 -- 典型的 full retrain 过拟合信号。

8.3 EST 参数 (models.py)

参数列表与 RSF 完全相同。区别不在参数，在算法：

RSF：每个节点从 sqrt(n) 个候选特征中搜索最优分裂点（贪心搜索）

EST：每个节点从 sqrt(n) 个候选特征中随机选分裂点（不搜索最优）

EST 单棵树更弱，但集成后方差更低，和 RSF 组合提供真正的模型多样性。

8.4 GBSA 参数 (models.py)

参数值含义
n_estimators300提升轮数（比 RSF 多，因为每轮只加一棵浅树）
max_depth3每棵树深度，比 RSF 浅（梯度提升用浅树叠加）
learning_rate0.02学习率，每棵新树的贡献缩小到 2%，防止过拟合
subsample0.8每轮随机抽 80% 样本训练（随机梯度提升）
dropout_rate0.1每轮随机丢弃 10% 已有的树（类似 DART 策略）
min_samples_leaf8叶节点最少 8 个样本
min_samples_split16分裂最少 16 个样本

已失败 实验 #10 证明 GBSA 不适合本任务：12h 概率被压缩到 0.212-0.268 的极窄范围，C-Index 对这种"近似平局"极其敏感，导致分数崩溃。
